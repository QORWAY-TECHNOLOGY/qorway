## Core System Architecture

QORWAY operates as a five-layer Decision Intelligence infrastructure:

```text
1. Atlas         → Causal Intelligence Engine
2. Domain Packs  → Context & Business Intelligence Layer
3. PolicyCore    → Constraint Validation Layer
4. PulseFlow     → Execution Orchestration Layer
5. GreenCore     → Execution Optimization Layer
```

These layers are supported by the Knowledge Graph Infrastructure, which provides system memory, causal structure, decision lineage, dependency visibility, and feedback integration.

---

## Layer Summary

### Atlas — Causal Intelligence Engine

Atlas reasons over structured organizational reality and produces decision candidates, causal chains, impact paths, and explanation structures.

Atlas proposes. It does not execute.

### Domain Packs — Context Intelligence Layer

Domain Packs make QORWAY domain-aware.

They describe how a domain works structurally at a public-safe level: finance, supply chain, CSRD/RSE, regulator, workforce capital, sovereign resilience, or other enterprise domains.

Domain Packs contextualize. They do not override constraints.

### PolicyCore — Constraint Validation Layer

PolicyCore validates whether a proposed decision may move forward.

It evaluates regulatory, financial, ESG, operational, sovereignty, human oversight, and tenant-specific constraints.

No decision becomes executable unless PolicyCore validates it.

### PulseFlow — Execution Orchestration Layer

PulseFlow transforms validated decisions into governed execution flows.

It provides event-driven orchestration, execution traceability, and feedback capture.

No execution should happen outside PulseFlow.

### GreenCore — Execution Optimization Layer

GreenCore optimizes approved execution paths across locality, latency, cost, energy, carbon, and sovereignty boundaries.

GreenCore cannot override PolicyCore.

## Knowledge Graph Infrastructure

QORWAY uses Knowledge Graph Infrastructure as the memory and causal structure layer.

It supports two graph levels:

```text
Universal Knowledge Graph
Private Tenant Knowledge Graph
```

The Universal Knowledge Graph contains reusable, non-identifiable domain structures.

The Private Tenant Knowledge Graph represents the sovereign causal reality of one organization.

Core rule:

> One tenant's graph must never become another tenant's intelligence.
